I was unable to find a good, clear explanation of what all of these terms mean, despite them being used in documentation often.

After consulting multiple sources, it turns out many of these terms are, in fact, ambiguous, and used to mean different things in different contexts.
I found this to be quite surprising, given the precise nature of computers and software development.

I treated the problem as if I was creating a function that accepts a string as input, representing the URL, and provides an object with property accessors as output.  
This forces the terms to be explicitly defined with a 1:1 relationship between term and meaning.
The javascript `URL` API provides part of this, but omits many common terms.

Some of the sources I referenced are:

https://www.icann.org/en/icann-acronyms-and-terms

https://web.dev/articles/url-parts

https://www.rfc-editor.org/rfc/rfc8499.html

https://developer.mozilla.org/en-US/docs/Web/API/URL

https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Web_mechanics/What_is_a_URL#anchor
